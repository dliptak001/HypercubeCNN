# HypercubeCNN – Architecture for Dummies

Pure C++ (zero dependencies).  
Scalar activations only.  
Elegant simplicity is our guiding principle.

## 1. One channel = one hypercube
- The hypercube has exactly **N = 1 << DIM** vertices.  
- Each vertex holds **one scalar float** (the activation).  
- Therefore one channel = a flat array of exactly **N floats**.

This is the base unit.

## 2. C channels = C independent hypercubes
- Each of the C channels is a complete, separate copy of the hypercube geometry.  
- They live side-by-side in memory as **one contiguous float array of size C × N**.  
- Memory layout (channel-major):

```cpp
activations[ c * N + v ] = activation of channel c at vertex v
```

Example (DIM=3 → N=8):
- 1 channel  →  8 floats
- 16 channels → 128 floats (one big array)

The channels share the **exact same vertex set and Hamming-shell topology** but each represents a distinct learned feature — exactly like channels in a classic CNN.

## 3. The shared kernel
The kernel is simply a **fixed-size array of learnable floats**, one entry per Hamming distance we care about.

```cpp
// Example: radius r = 2
float kernel[3];   // kernel[0], kernel[1], kernel[2]
```

- `kernel[d]` = the single shared weight that multiplies **every** neighbor at exact Hamming distance `d` from the center vertex.
- `d = 0` is the center vertex itself (self-connection — optional but useful).
- We never need one weight per individual neighbor — the hypercube’s vertex-transitive symmetry makes all positions at the same distance mathematically identical.

For a typical early layer we use `r = 1` (2 floats) or `r = 2` (3 floats). Extremely parameter-efficient.

## 4. Applying the kernel (the convolution step)
For every output channel `c_out` and every vertex `v`:

```cpp
out[c_out][v] = 0;
for each input channel c_in
    for d = 0 to r
        sum_of_shell = 0;
        for every neighbor u where Hamming_distance(v, u) == d
            sum_of_shell += in[c_in][u];
        out[c_out][v] += kernel[d] * sum_of_shell;
```

Then apply the non-linearity (ReLU, etc.) to `out[c_out][v]`.

The inner sum over a shell is trivial in C++ (generated by flipping exactly `d` bits via XOR). No adjacency lists, no padding, no border cases — every vertex is treated identically.

## 5. Data flow through one conv layer (C_in → C_out)
- Input array size:  `C_in  * N` floats
- Output array size: `C_out * N` floats

Each output channel is produced by applying the same tiny shared kernel across **all** input channels at every vertex.  
The channels are strongly coupled only during this convolution step (exactly as in a classic CNN).

## 6. Stacking layers
- Layer 1 output array (`C_out1 * N`) becomes **directly** the input array for Layer 2 (`C_in2 * N`).
- No reshaping, no copying, no mapping. Just pass the pointer and the numbers `C_in2`, `C_out2`, `DIM`.
- The spatial size (number of vertices) **never changes** until we explicitly add pooling (subcube average/max or dimension reduction).

Concrete numbers (DIM=3, N=8):
- Layer 1: 1×8 → 16×8 = 128 floats
- Layer 2: 16×8 → 32×8 = 256 floats
- Layer 3: 32×8 → 64×8 = 512 floats

The hypercube geometry stays fixed; only the depth (number of channels) grows.

---

This is the complete, minimal picture.  
No abstractions.  
No extra layers.  
Ready for direct implementation in zero-dependency C++.
